Pages — Notebook to Route Mapping

Each notebook maps to one primary route. Shared state flows through Svelte stores.

Run/session browser with filter + sort. Inline description editing. Aggregation management. Select entities → selectionStore → navigate to dashboard.

File picker + SHA dedup. Profile selector (named profiles in localStorage). Per-file progress. Gzip upload → binary response. Writes artifacts to IndexedDB on completion. Run description prompt inline.

Reads selectionStore (sessions + aggregations). Histograms + scatter charts from events/metrics tables in IndexedDB. Engineering units toggle (mm / normalised). Selection persists across page navigation.
NOTEBOOK
session_test_notebook
Time-series browser,
event annotations
→
/session/[id] Session Browser
Lazy-loads signal Float32Arrays from IndexedDB on open. Zoomable time-series (Plotly.js). Event markers overlaid. Window bookmarks. Clicking an event → jumps to that window.
NOTEBOOK
event_schema_test_harness
YAML editor, single CSV,
event preview
→
/schema Schema Editor (dev/power-user tool)
YAML editor (CodeMirror). Upload a single test CSV → POST /api/validate-schema → live event preview table. Save schema to localStorage. Iterate without affecting the main library.
Shared infrastructure across all pages
selectionStore
Persisted to localStorage. Drives dashboard + session browser reactively.
libraryStore
Run/session index from localStorage. Updated after each preprocess job.
Global nav
Library → Preprocess → Dashboard → Session → Schema. Current selection always visible.
